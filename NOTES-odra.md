## Module struct + impl

```rust
#[odra::module]
pub struct Flipper {
    value: Var<bool>,
}

#[odra::module]
impl Flipper {
    pub fn init(&mut self) {
        self.value.set(false);
    }
    pub fn set(&mut self, value: bool) { self.value.set(value); }
    pub fn flip(&mut self) { self.value.set(!self.get()); }
    pub fn get(&self) -> bool { self.value.get_or_default() }
}
```

Both the struct and the impl carry `#[odra::module]`. Every `pub` fn in the impl becomes a
callable entry point. `init` is the one-time constructor.

## Storage types and their methods

- `Var<T>`: `set(value)`, `get() -> Option<T>`, `get_or_default() -> T` (T: Default).
- `Mapping<K, V>`: `set(&key, value)`, `get(&key) -> Option<V>`, `get_or_default(&key) -> V`.
- `List<T>`: `push(value)`, `len() -> u32`, `iter()`. NOTE: no indexed `.get(idx)`. Iterate with
  `iter()`. For our finalize/resolve loops, iterate facet ids via `self.facet_ids.iter()`, not
  index access.
- `SubModule<M>`: nested module composition (not needed for v1).

## Custom types and errors

```rust
#[odra::odra_type]
pub struct Dog { pub name: String, pub age: u8, pub owner: Option<Address> }
```

`#[odra::odra_type]` gives serialization + schema. Enums use the same attribute. For errors, use
`#[odra::odra_error]` on an enum (confirm exact spelling from the scaffold; some versions use
`execution_error!`). Revert with `self.env().revert(Error::Variant)`.

## Tests (HostEnv)

```rust
use odra::{host::Deployer, prelude::*};

let env = odra_test::env();                       // HostEnv
let mut c = MyModule::deploy(&env, MyModuleInitArgs { /* ... */ });
env.set_caller(env.get_account(0));
let acct = env.get_account(1);
c.some_entry_point(args);                          // call pub fns directly
assert_eq!(c.some_view(), expected);
```

Odra generates `{Module}HostRef` and `{Module}InitArgs` per module. For a no-arg `init(&mut self)`,
deploy takes `NoArgs` (confirm against the scaffold: it may be `MyModuleInitArgs` with no fields or
`odra::host::NoArgs`).
